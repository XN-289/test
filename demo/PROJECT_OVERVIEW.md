# ViaSurg AI CMO - 项目完整说明

## 🎯 项目概述

ViaSurg AI CMO 是一个 AI 驱动的智能营销引擎，专为医疗设备 B2B 出海营销设计。系统通过 4 个核心 Skills 的协同工作，实现从市场信号收集到内容生成的全流程自动化。

### 核心价值主张

**"不与巨头拼未来，用精准数据截流现在"**

- 精准截流：通过 SKU 级关键词分析，精准截取竞争对手流量
- 成本优势：利用长尾关键词和精准定位，降低获客成本
- 自动化：减少人工干预，实现内容生产的自动化
- 可扩展：系统架构支持快速扩展到其他医疗品类

## ⚡ 核心 Skills 详解

### 1. Strategy Operator（市场情报分析师）

**职责**：维护战略图谱，通过 OODA 循环处理市场信号

**OODA 循环**：
- **Observe**：收集市场信号
- **Orient**：分析信号，识别机会
- **Decide**：生成应对策略
- **Act**：更新战略图谱

**输入**：
- 市场信号（关键词数据、竞品动态）
- 历史策略
- 反馈数据

**输出**：
- 更新后的战略图谱
- 新的应对策略

**关键功能**：
- 信号强度分析
- 商业价值评估
- 策略优先级排序
- 竞品动态监控

### 2. Wiki Compiler（产品知识工程师）

**职责**：构建和维护领域本体图谱，将原始文档编译为结构化知识

**工作流程**：
1. 读取原始文档
2. 提取实体和关系
3. 更新知识图谱
4. 生成双向链接

**输入**：
- 原始文档（产品资料、技术文档）
- 战略意图

**输出**：
- 更新后的知识图谱
- 结构化实体文件

**关键功能**：
- 实体提取（产品、认证、竞品）
- 关系识别（替代、对比、认证）
- 图谱更新
- 索引生成

### 3. Site Generator（内容发布专家）

**职责**：将知识图谱转换为公开的、SEO 优化的网站

**工作流程**：
1. 加载设计规范
2. 渲染页面组件
3. 生成 Schema 标记
4. 质量检查

**输入**：
- LLM Wiki（实体、图谱）
- 设计配置

**输出**：
- SEO 优化的网页
- JSON-LD Schema
- sitemap.xml

**关键功能**：
- 组件化渲染
- SEO 优化
- Schema 生成
- 质量验证

### 4. Autoresearch（智能优化引擎）

**职责**：自主迭代改进系统，支持无人值守的运行

**迭代循环**：
1. **Review**：分析当前状态
2. **Ideate**：生成改进假设
3. **Modify**：执行单点修改
4. **Verify**：验证改进效果
5. **Decide**：保留或回滚

**输入**：
- 优化目标
- 验证命令
- 迭代范围

**输出**：
- 优化后的内容
- 迭代日志
- 经验教训

**关键功能**：
- 自主迭代
- 质量验证
- 经验学习
- 自动回滚

## 🔄 完整工作流程

### 阶段一：信号收集

```
输入：市场情报、竞品动态、关键词数据
处理：信号分类、优先级排序、商业价值评估
输出：结构化信号数据
```

**示例**：
```json
{
  "signal": "endoscopic clip CPC 高达 $60",
  "source": "Google Keywords Planner",
  "priority": "high",
  "commercial_value": "高转化率商业机会"
}
```

### 阶段二：战略分析

```
输入：结构化信号数据
处理：OODA 循环分析、策略生成
输出：应对策略
```

**示例**：
```json
{
  "strategy": "SKU 截流战略",
  "target": "Ethicon CDH29P",
  "action": "创建替代品页面",
  "expected_outcome": "获取 30% 竞品流量"
}
```

### 阶段三：知识编译

```
输入：原始文档、战略意图
处理：实体提取、关系识别、图谱更新
输出：结构化知识库
```

**示例**：
```json
{
  "entity": "内镜止血夹",
  "type": "product",
  "specifications": ["3mm", "5mm", "7mm"],
  "certifications": ["CE", "ISO 13485"],
  "competitors": ["QuickClip2", "TriClip"]
}
```

### 阶段四：内容生成

```
输入：知识图谱、设计配置
处理：内容渲染、SEO 优化、Schema 生成
输出：营销页面
```

**示例**：
```html
<title>Endoscopic Clips - ViaSurg | 40% Cost Savings</title>
<meta name="description" content="CE/FDA certified endoscopic clips...">
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "ViaSurg Endoscopic Clips"
}
</script>
```

### 阶段五：质量验证

```
输入：生成的营销页面
处理：HTML 验证、Schema 检查、SEO 评分
输出：质量报告
```

**检查项**：
- HTML 结构完整性
- Schema.org 语法正确性
- SEO 元素完整性
- 页面加载速度
- 移动端适配性

## 🎯 用户场景

### 场景一：经销商寻找供应商

**用户**：美国医疗器械经销商采购经理

**需求**：寻找高性价比的内镜止血夹供应商

**流程**：
1. 搜索 "endoscopic clip supplier"
2. 看到 ViaSurg 广告："40% Cost Savings"
3. 点击进入产品页面
4. 查看产品规格、认证信息、价格对比
5. 填写询价表单
6. 收到报价和样品
7. 确认订单

**系统动作**：
- Strategy Operator：识别高价值关键词
- Wiki Compiler：提取产品信息
- Site Generator：生成 SEO 优化页面
- Autoresearch：持续优化页面转化率

### 场景二：医院采购主任降低成本

**用户**：中小型医院采购主任

**需求**：降低手术器械采购成本

**流程**：
1. 搜索 "biopsy forceps price comparison"
2. 看到 ViaSurg 对比文章
3. 了解 ViaSurg 产品优势
4. 下载白皮书
5. 联系销售代表
6. 获得批量采购报价
7. 签订长期合作协议

**系统动作**：
- Strategy Operator：分析价格敏感度
- Wiki Compiler：生成对比内容
- Site Generator：创建 ROI 计算器
- Autoresearch：优化转化漏斗

### 场景三：科室主任寻找替代品

**用户**：大型医院内镜科室主任

**需求**：寻找 Ethicon CDH29P 的替代品

**流程**：
1. 搜索 "cdh29p alternative"
2. 看到 ViaSurg "Alternatives to Ethicon CDH29P" 页面
3. 查看技术参数对比
4. 了解临床数据
5. 申请样品试用
6. 评估试用结果
7. 决定采购

**系统动作**：
- Strategy Operator：识别 SKU 截流机会
