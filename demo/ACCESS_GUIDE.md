## 🎮 系统功能

### 1. 控制台（Dashboard）

- **位置**：默认首页
- **功能**：
  - 实时监控系统状态
  - 查看关键指标（信号数、实体数、页面数、策略数）
  - 工作流可视化
  - 系统日志

### 2. 战略推演官（Strategy Operator）

- **位置**：侧边栏 → 🎯 战略推演官
- **功能**：
  - 输入市场信号
  - 执行 OODA 循环
  - 生成应对策略
  - 更新战略图谱

### 3. 本体架构师（Wiki Compiler）

- **位置**：侧边栏 → 📚 本体架构师
- **功能**：
  - 处理原始文档
  - 提取实体和关系
  - 更新知识图谱
  - 生成双向链接

### 4. 全栈站长（Site Generator）

- **位置**：侧边栏 → 🌐 全栈站长
- **功能**：
  - 渲染 SEO 优化的网页
  - 生成 JSON-LD Schema
  - 更新 sitemap.xml
  - 质量检查和自动纠错

### 5. 自动研究（Autoresearch）

- **位置**：侧边栏 → 🔄 自动研究
- **功能**：
  - 自主迭代改进
  - 质量验证
  - 经验学习
  - 无人值守运行

### 步骤 1：查看控制台

1. 打开 http://localhost:3000

2. 点击 **"执行 OODA 循环"** 按钮
3. 查看执行结果

在 `test_materials` 目录中查看：
- `market_signals.json`：市场信号数据

## 🔧 故障排除

### 问题：无法访问 http://localhost:3000

**解决方案**：
1. 检查服务器是否运行：
   ```bash
   cd d:/test/demo
   node server.js
   ```

2. 检查端口是否被占用：
   ```bash
   netstat -ano | findstr :3000
   ```

3. 如果端口被占用，修改 `server.js` 中的 `PORT` 变量

### 问题：页面显示不正常

**解决方案**：
1. 清除浏览器缓存
2. 尝试使用其他浏览器
3. 检查浏览器控制台是否有错误

### 问题：功能不工作

**解决方案**：
1. 刷新页面重试
2. 查看控制台日志
3. 检查网络连接
